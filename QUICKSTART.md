# Socratic Writer — Quickstart (v0 Demo)

You now have a real, working version of the tool.

## How to Start

1. **Easiest**: Double-click the **Socratic Writer** icon on your Desktop.
2. Or in terminal: `python3 ~/writer/launch.py`

A clean Firefox window will open with the writing surface.

**Pro immersion tip**: Once it opens, press **F11** for true fullscreen. The mouse cursor will also auto-hide after a couple seconds of typing.

## The Rules (Enforced by the App)

- You can only type forward.  
  **Backspace, Delete, arrow keys, Ctrl+Z, paste in the middle — all do nothing.**

- The **only** extra control is the **space bar**:
  - Type normally (spaces between words work as expected).
  - When you reach the end of a thought/paragraph and want reflection:
    → **Press and hold the space bar for ~650 milliseconds**.
  - Socratic questions (generated only from what you have written) appear in the right margin.

- You can completely ignore the questions and keep typing. They will gently fade if you write a lot more.

## Instant Demo Texts

While the surface is focused, type one of these then immediately hold space:

- `sample1` → First-thoughts example (great for seeing the Socratic engine)
- `sample2` → Relationship tension example
- `sample3` → Creative block example

These load beautiful seed writing so you can experience the questions without writing first.

## Other Hidden Commands (while surface is focused)

- `export` + hold space → Downloads your current writing as a clean .txt file
- `clearall` + hold space → Wipes everything (with confirmation)

## What Makes This Different

This is not a normal text editor. It is deliberately hostile to editing, perfectionism, and distraction.

The goal is **forward motion + occasional deep reflection** triggered only when *you* decide you want it.

## Current Limitations (Honest)

- Socratic questions are generated by Grok (via the local Grok Build CLI) when available, with a sharp local heuristic as instant fallback.
- If the Grok CLI is not reachable in the environment, you still get excellent questions from the built-in engine with zero difference in the writing experience.
- Persistence is per-browser (localStorage). Use `export` regularly if you want files.
- Only one writing surface at a time in v0.
- The visual design is intentionally very quiet. It will get even quieter over time.
